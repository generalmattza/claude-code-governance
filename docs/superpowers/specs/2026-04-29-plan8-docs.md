# Plan 8 - Documentation Track Spec

> Status: 2026-04-29.

## Goal

Ship full Track 1-5 documentation for `claude-code-security` and tag `v0.8.0-rc.0` (release candidate). Plan 9 (release engineering) and Plan 10 (pilot validation) remain before `v1.0.0`.

## Tracks

- **Track 1 - Project README:** entry point, ToC, badges, install channels, profile chooser, honesty box.
- **Track 2 - Deployment guides:** `docs/deployment/mdm-jamf.md` (existing, macOS), plus substantive Windows and Linux MDM guides at `installers/windows/README.md` and `installers/linux/README.md`.
- **Track 3 - Operator reference:** `docs/threat-model.md`, auto-generated `docs/coverage-matrix.md`, auto-generated per-hook docs at `docs/hooks/<name>.md` for all 26 hooks.
- **Track 4 - Configuration reference:** `docs/settings-reference.md` covering every settings.json key the project uses, plus the README profile chooser table.
- **Track 5 - Project meta:** ADRs (existing 0001-0004 plus new 0005, 0006), CHANGELOG.md, SECURITY.md, LICENSE, CODE_OF_CONDUCT.md.

## Unique value-add

`docs/known-bypasses.md` documents detection gaps openly. This is the honest signal that distinguishes this repo from a "catches everything" claim. Eight bypasses documented, each with vector / detection status / recommended response.

## Auto-generators

Two Node ESM scripts under `scripts/`:

1. `scripts/gen-hook-docs.mjs` walks `packages/hooks/src/*/index.ts`, imports each compiled module from `dist/`, reads its manifest, and writes `docs/hooks/<name>.md`.
2. `scripts/gen-coverage-matrix.mjs` walks all manifests, groups by threat ID, writes `docs/coverage-matrix.md`.

Both expose pnpm scripts (`gen:hook-docs`, `gen:coverage-matrix`). For Plan 8 they run locally; CI enforcement is deferred to Plan 9.

## Constraints

- No em dashes, no Claude footers.
- Plan 1-7 tests still pass (313).
- Coverage stays above 90 percent.
- Auto-gen scripts are JavaScript ESM (no TypeScript) for simplicity.

## Out of Scope

- Hand-curated notes/limitations per hook (deferred; auto-gen leaves a TODO line).
- Full Windows / Linux installer implementations (templates only; v1.1 / v1.2).
- CI gate that fails on doc drift (deferred to Plan 9 release engineering).
